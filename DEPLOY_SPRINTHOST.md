# Инструкция: заливка на SprintHost

## 1) Подготовка проекта

1. (Рекомендуется) Скопируйте `.env.production.example` в `.env.production.local` и укажите `NEXT_PUBLIC_ASSISTANT_API_URL` на **ваш** публичный API (Павлик). Без этого во вкладке **Павлик** в собранном сайте останется адрес `localhost`.
2. Выполните в папке проекта:

```bash
npm install
npm run hosting:build
```

`hosting:build` выполняет `next build` и скрипт для Apache: папка `_next` в `out` переименовывается в `next-assets`, в корне и у ассетов появляются служебные `.htaccess` (на части хостингов иначе сломается подгрузка JS/CSS).

Альтернативы: `npm run export:shared-hosting` (то же), или только `npm run build` + ручная заливка, если на вашем тарифе `_next` не блокируется.

После успешного шага появляется папка `out` — её содержимое копируется на хостинг.

### Если сайт будет в подпапке (не в корне домена)

Пример: сайт открывается как `https://example.com/vss/`

Перед сборкой задайте переменную:

```bash
# PowerShell
$env:NEXT_PUBLIC_BASE_PATH="vss"
npm run hosting:build
```

Для корня домена переменная не нужна.

## 2) Подключение домена в SprintHost

1. Зайдите в панель SprintHost.
2. Откройте нужный сайт (домен/поддомен).
3. Перейдите в файловый менеджер в web-каталог сайта (обычно `public_html` или `www`).

## 3) Загрузка файлов

1. Удалите старые файлы сайта (если они не нужны).
2. Загрузите **содержимое** папки `out` в web-каталог.
3. Убедитесь, что в корне лежит `index.html` и папка `handbook`.

> Важно: загружайте содержимое `out`, а не папку `out` целиком.

## 4) Проверка

Проверьте:

- `https://ваш-домен/`
- `https://ваш-домен/handbook/`

## 5) Обновления

При каждом изменении проекта:

1. `npm run hosting:check` (по желанию — линт + пробная сборка)
2. `npm run hosting:build` (с актуальным `.env.production.local`, если менялся API или подпапка)
3. Перезалить **содержимое** `out` с заменой файлов, не забыв про `.htaccess` в корне `out`.

## Примечание

Проект уже подготовлен под статический хостинг (без запуска Node.js на сервере), поэтому подходит для стандартного тарифа SprintHost.
