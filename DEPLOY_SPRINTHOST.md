# Инструкция: заливка на SprintHost

## 1) Подготовка проекта

Выполните локально в папке проекта:

```bash
npm install
npm run export
```

После сборки появится папка `out` — это готовая версия сайта для хостинга.

### Если сайт будет в подпапке (не в корне домена)

Пример: сайт открывается как `https://example.com/vss/`

Перед сборкой задайте переменную:

```bash
# PowerShell
$env:NEXT_PUBLIC_BASE_PATH="vss"
npm run export
```

Для корня домена переменная не нужна.

## 2) Подключение домена в SprintHost

1. Зайдите в панель SprintHost.
2. Откройте нужный сайт (домен/поддомен).
3. Перейдите в файловый менеджер в web-каталог сайта (обычно `public_html` или `www`).

## 3) Загрузка файлов

1. Удалите старые файлы сайта (если они не нужны).
2. Загрузите **содержимое** папки `out` в web-каталог.
3. Убедитесь, что в корне лежит `index.html` и папка `handbook`.

> Важно: загружайте содержимое `out`, а не папку `out` целиком.

## 4) Проверка

Проверьте:

- `https://ваш-домен/`
- `https://ваш-домен/handbook/`

## 5) Обновления

При каждом изменении проекта:

1. `npm run export`
2. Перезалить содержимое `out` на хостинг с заменой файлов.

## Примечание

Проект уже подготовлен под статический хостинг (без запуска Node.js на сервере), поэтому подходит для стандартного тарифа SprintHost.
